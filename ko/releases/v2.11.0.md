# Excelize v2.11.0

* 출시 예정일: 2026 년 7 월 7 일

## 릴리스 노트

이 릴리스에서 가장 주목할 만한 변경 사항은 다음과 같습니다:

### 하위 호환성 변경 사항

* 요구 사항 업데이트: 종속성 패키지 `golang.org/x/net` 업그레이드로 인해 Go 언어 버전 1.25.0 이상이 필요합니다
* `AddChart`, `AddChartSheet`, `AddShape` 함수의 하위 호환성 변경 사항
  * `Chart`의 `Title` 필드 데이터 형식을 `[]RichTextRun`에서 `ChartTitle`로 변경
  * `Shape`의 `Line` 필드 데이터 형식을 `ShapeLine`에서 `LineOptions`로 변경
  * 데이터 형식 `ChartDashType`을 `LineDashType`으로 이름 변경
  * 데이터 형식 `ChartLineType`을 `LineType`으로 이름 변경
  * 데이터 형식 `ChartLine`을 `LineOptions`로 이름 변경

### 보안

* 무한 메모리 할당 방지를 위해 워크시트 읽기 시 행 번호 검사 (CVE-2026-54063), 관련 보안 권고사항 [#GHSA-h69g-9hx6-f3v4](https://github.com/xuri/excelize/security/advisories/GHSA-h69g-9hx6-f3v4)
* 워크시트 행 번호 오버플로 최대 제한 유효성 검사를 통해 공격자가 제어하는 메모리 할당 방지 (CWE-400 및 CWE-770), 관련 보안 권고사항 [#GHSA-q5j5-6p94-4gwc](https://github.com/xuri/excelize/security/advisories/GHSA-q5j5-6p94-4gwc)
* 공유 문자열 테이블의 잘못된 인덱스로 셀 값 가져오기 시 패닉 방지 (CWE-248 및 CWE-755), 관련 보안 권고사항 [#GHSA-fx5j-qcqg-grpf](https://github.com/xuri/excelize/security/advisories/GHSA-fx5j-qcqg-grpf)

### 새로운 기능

* 새로운 데이터 형식 `ChartTitle` 추가
* 새로운 데이터 형식 `PivotTableShowValuesAs` 추가
* 새로운 열거형 `PivotTableShowValuesAsType` 추가
* 3개의 새로운 내보내기 오류 변수 추가: `ErrPivotTableShowValuesAsBaseField`, `ErrPivotTableShowValuesAsBaseItem`, `ErrUnsupportedPivotTableShowValuesAsType`
* 데이터 형식 `PivotTableField`에 새로운 필드 `ShowValuesAs` 추가
* 데이터 형식 `PivotTableOptions` 및 `SlicerOptions`에 새로운 필드 `SelectedItems` 추가
* 열 너비 자동 맞춤을 지원하는 새로운 함수 `AutoFitColWidth` 추가, 관련 이슈 [#92](https://github.com/xuri/excelize/issues/92)
* `SetSheetName` 함수가 유효하지 않은 원본 이름의 시트 이름 변경을 허용하지 않던 문제 수정, 관련 이슈 [#548](https://github.com/xuri/excelize/issues/548)
* `CalcCellValue` 함수에서 물결표 와일드카드 기준 지원
* `CalcCellValue` 함수에서 암시적 교집합을 지원하고 정규식 기준 앵커링 수정
* `CalcCellValue` 함수에서 시트 범위에 걸친 3D 참조 지원, 관련 이슈 [#2303](https://github.com/xuri/excelize/issues/2303)
* `CalcCellValue` 함수에서 셀 원시 값과 함께 수식 계산 결과 반환 지원, 관련 이슈 [#2316](https://github.com/xuri/excelize/issues/2316)
* `AddPivotTable` 함수에서 동일한 데이터 필드가 피벗 테이블의 열, 행 및 필터 필드에 동시에 나타나는지 여부를 검사하여 손상된 통합 문서 생성 방지
* `AddPivotTable` 및 `GetPivotTables` 함수에서 피벗 테이블 데이터 필드의 "값 표시 형식" 설정 및 가져오기 지원, 관련 이슈 [#2340](https://github.com/xuri/excelize/issues/2340)
* `AddPivotTable`, `AddSlicer`, `GetPivotTables`, `GetSlicers` 함수에서 피벗 테이블 및 피벗 테이블 슬라이서의 선택된 항목 설정 및 가져오기 지원, 관련 이슈 [#2154](https://github.com/xuri/excelize/issues/2154)
* `AddChart` 및 `AddChartSheet` 함수에서 수식을 사용한 차트 제목 설정 지원
* `AddChart` 및 `AddChartSheet` 함수에서 차트 제목 및 차트 축 제목의 레이아웃 설정 지원
* `AddChart` 및 `AddChartSheet` 함수에서 차트 제목 및 차트 축 제목의 선 서식 설정 지원
* `AddShape` 함수에서 도형의 선 서식 설정 지원
* `NewStyle` 함수에서 사용자 지정 색상 값이 지정되지 않은 경우 단색 채우기 스타일 생성 시 전경색 및 배경색 자동 설정 지원

### 호환성 개선

* 차트 시트에서 Excel Online 및 KingSoft WPS&trade; Office 에 차트가 표시되지 않는 문제 수정, 관련 이슈 [#2273](https://github.com/xuri/excelize/issues/2273)

### 버그 수정

* v2.8.1 버전에서 도입된 회귀 문제 수정 [#1383](https://github.com/xuri/excelize/issues/1383)
* 채우기 시리즈가 없는 차트 추가 시 `AddChart` 함수에서 패닉이 발생하는 문제 수정
* 일부 경우 잘못된 행 참조 시 `CalcCellValue` 함수에서 패닉이 발생하는 문제 수정, 이슈 [#2266](https://github.com/xuri/excelize/issues/2266)
* `CalcCellValue` 함수 사용 시 수식의 정확한 텍스트 일치가 부분 문자열과 일치하지 않는 문제 수정
* `CalcCellValue` 함수에서 수식 함수 BINOM.INV, CHIINV, CHISQ.DIST, IMSECH, IMSQRT 및 NEGBINOM.DIST의 계산 정확도 문제 수정
* 일부 경우 `CalcCellValue` 함수가 FREQUENCY 의 잘못된 계산 결과를 반환하는 문제 수정
* 괄호 또는 연산자 내에 오류 값이 있을 때 `CalcCellValue` 함수가 잘못된 계산 결과를 반환하는 문제 수정, 이슈 [#2344](https://github.com/xuri/excelize/issues/2344)
* 작성자가 이미 작성자 목록에 존재하는 경우 `AddComment` 함수가 모든 메모에 잘못된 작성자 ID를 할당하는 문제 수정, 이슈 [#2289](https://github.com/xuri/excelize/issues/2289)
* 오류 반환 전 파일 잠금 해제기가 호출되지 않아 발생할 수 있는 교착 상태 수정
* 쉼표 배율이 포함된 사용자 지정 숫자 서식이 올바르게 적용되지 않는 문제 수정, 이슈 [#2297](https://github.com/xuri/excelize/issues/2297)
* 열 반복자를 통해 잘못된 워크시트 XML이 포함된 손상된 통합 문서를 읽을 때 패닉이 발생하는 문제 수정, 이슈 [#2299](https://github.com/xuri/excelize/issues/2299)
* 외부 참조가 포함된 수식 조정 후 작은따옴표가 누락되는 문제 수정
* 일부 경우 ECMA-376 Agile 암호화 및 SHA512 해시 알고리즘으로 보호된 통합 문서의 복호화에 실패하는 문제 수정, 이슈 [#2328](https://github.com/xuri/excelize/issues/2328)
* `SetCellFormula` 함수가 일반 수식으로 덮어쓸 때 공유 수식 메타데이터를 지우지 않는 문제 수정, 이슈 [#2337](https://github.com/xuri/excelize/issues/2337)

### 성능 최적화

* 암호화되지 않은 통합 문서 읽기 시 메모리 사용량 감소, 행 반복자 메모리 사용량 최대 85% 감소
* 좌표 변환 및 일부 내부 함수의 성능 최적화
  * `ColumnNumberToName` 함수의 메모리 할당 약 90% 감소
  * `CoordinatesToCellName` 함수의 메모리 할당 약 13% 감소
  * 내부 함수 `isNumeric`의 메모리 할당 약 68% 감소

### 기타

* Go Modules 는 모듈 업데이트에 의존합니다
* 단위 테스트 및 godoc 업데이트
* 다국어를 지원하는 [문서 웹사이트](https://xuri.me/excelize): 아랍어, 독일어, 영어, 스페인어, 프랑스어, 이탈리아어, 일본어, 한국어, 포르투갈어, 러시아어, 중국어 간체 및 중국어 번체, 업데이트됨
* [excelize-wasm](https://github.com/xuri/excelize-wasm) WebAssembly / JavaScript 지원을 위한 NPM 패키지 릴리스 업데이트
* [excelize](https://github.com/xuri/excelize-py) Python 을 위한 PyPI 패키지 릴리스 업데이트
* [ExcelizeCs](https://github.com/xuri/excelize-cs) C# 을 위한 NuGet .Net 패키지 릴리스 업데이트
