# Excelize v2.11.0

* リリース予定: 2026年7月7日

## リリースノート

このバージョンでの最も注目すべき変更点は次のとおりです:

### 互換性のヒント

* 要件更新：依存パッケージ `golang.org/x/net` のアップグレードに伴い、Go 言語バージョン 1.25.0 以上が必要になりました
* `AddChart`、`AddChartSheet`、`AddShape` 関数の破壊的変更
  * `Chart` の `Title` フィールドのデータ型を `[]RichTextRun` から `ChartTitle` に変更
  * `Shape` の `Line` フィールドのデータ型を `ShapeLine` から `LineOptions` に変更
  * データ型 `ChartDashType` を `LineDashType` に改名
  * データ型 `ChartLineType` を `LineType` に改名
  * データ型 `ChartLine` を `LineOptions` に改名

### セキュリティ

* 無制限のメモリ割り当てを防ぐため、ワークシート読み取り時に行番号を検証するようになりました (CVE-2026-54063)、関連するセキュリティアドバイザリ [#GHSA-h69g-9hx6-f3v4](https://github.com/xuri/excelize/security/advisories/GHSA-h69g-9hx6-f3v4)
* ワークシートの行番号がオーバーフローの最大制限を超えていないことを検証することで、攻撃者が制御するメモリ割り当てを防止 (CWE-400 および CWE-770)、関連するセキュリティアドバイザリ [#GHSA-q5j5-6p94-4gwc](https://github.com/xuri/excelize/security/advisories/GHSA-q5j5-6p94-4gwc)
* 共有文字列テーブルの無効なインデックスでセル値を取得する際のパニックを防止 (CWE-248 および CWE-755)、関連するセキュリティアドバイザリ [#GHSA-fx5j-qcqg-grpf](https://github.com/xuri/excelize/security/advisories/GHSA-fx5j-qcqg-grpf)

### 新機能

* 新しいデータ型 `ChartTitle` を追加
* 新しいデータ型 `PivotTableShowValuesAs` を追加
* 新しい列挙型 `PivotTableShowValuesAsType` を追加
* 3 つの新しいエクスポートエラー変数を追加：`ErrPivotTableShowValuesAsBaseField`、`ErrPivotTableShowValuesAsBaseItem`、`ErrUnsupportedPivotTableShowValuesAsType`
* データ型 `PivotTableField` に新しいフィールド `ShowValuesAs` を追加
* データ型 `PivotTableOptions` と `SlicerOptions` に新しいフィールド `SelectedItems` を追加
* 列幅の自動調整をサポートする新しい関数 `AutoFitColWidth` を追加, 関連する問題 [#92](https://github.com/xuri/excelize/issues/92)
* `SetSheetName` 関数が無効なソース名のシートの名前変更を許可するよう修正, 関連する問題 [#548](https://github.com/xuri/excelize/issues/548)
* `CalcCellValue` 関数でチルダワイルドカード条件をサポート
* `CalcCellValue` 関数で暗黙的な共通部分をサポートし、正規表現条件のアンカリングを修正
* `CalcCellValue` 関数でシート範囲をまたぐ 3D 参照をサポート, 関連する問題 [#2303](https://github.com/xuri/excelize/issues/2303)
* `CalcCellValue` 関数でセルの生の値を含む数式計算結果の返却をサポート, 関連する問題 [#2316](https://github.com/xuri/excelize/issues/2316)
* `AddPivotTable` 関数で、同じデータフィールドがピボットテーブルの列・行・フィルターフィールドに同時に含まれているかどうかの検証をサポートし、破損したワークブックの生成を防止
* `AddPivotTable` と `GetPivotTables` 関数でピボットテーブルデータフィールドの「値の表示形式」の設定・取得をサポート, 関連する問題 [#2340](https://github.com/xuri/excelize/issues/2340)
* `AddPivotTable`、`AddSlicer`、`GetPivotTables`、`GetSlicers` 関数でピボットテーブルおよびピボットテーブルスライサーの選択済みアイテムの設定・取得をサポート, 関連する問題 [#2154](https://github.com/xuri/excelize/issues/2154)
* `AddChart` と `AddChartSheet` 関数で数式を使用したグラフタイトルの設定をサポート
* `AddChart` と `AddChartSheet` 関数でグラフタイトルおよびグラフ軸タイトルのレイアウト設定をサポート
* `AddChart` と `AddChartSheet` 関数でグラフタイトルおよびグラフ軸タイトルの線の書式設定をサポート
* `AddShape` 関数で図形の線の書式設定をサポート
* `NewStyle` 関数で単色塗りつぶしスタイルを作成する際に、カスタムカラー値が指定されていない場合の前景色・背景色の自動設定をサポート

### 互換性の向上

* グラフシートで Excel Online および KingSoft WPS&trade; Office にグラフが表示されない問題を修正, 関連する問題 [#2273](https://github.com/xuri/excelize/issues/2273)

### 問題の修正

* v2.8.1 のリグレッション問題を修正 [#1383](https://github.com/xuri/excelize/issues/1383)
* フィルシリーズなしのグラフを追加する際に `AddChart` 関数がパニックになる問題を修正
* 一部のケースで無効な行参照時に `CalcCellValue` 関数がパニックになる問題を修正, 関連する問題 [#2266](https://github.com/xuri/excelize/issues/2266)
* `CalcCellValue` 関数使用時に数式の完全一致テキストマッチングがサブ文字列にマッチしない問題を修正
* `CalcCellValue` 関数における数式関数 BINOM.INV、CHIINV、CHISQ.DIST、IMSECH、IMSQRT、NEGBINOM.DIST の計算精度の問題を修正
* 一部のケースで `CalcCellValue` 関数が FREQUENCY の計算結果を誤って返す問題を修正
* 括弧または演算子内にエラー値が含まれる場合に `CalcCellValue` 関数が誤った計算結果を返す問題を修正, 関連する問題 [#2344](https://github.com/xuri/excelize/issues/2344)
* 著者が既に著者リストに存在する場合に `AddComment` 関数がすべてのコメントに誤った著者 ID を割り当てる問題を修正, 関連する問題 [#2289](https://github.com/xuri/excelize/issues/2289)
* エラーを返す前にファイルアンロッカーが呼び出されないことによるデッドロックの可能性を修正
* カスタム数値書式でコンマスケーリングが正しく適用されない問題を修正, 関連する問題 [#2297](https://github.com/xuri/excelize/issues/2297)
* 列イテレーターで無効なワークシート XML を含む破損したワークブックを読み取る際にパニックが発生する問題を修正, 関連する問題 [#2299](https://github.com/xuri/excelize/issues/2299)
* 外部参照を含む数式を調整した後にシングルクォートが欠落する問題を修正
* 一部のケースで ECMA-376 Agile 暗号化と SHA512 ハッシュアルゴリズムで保護されたワークブックの復号に失敗する問題を修正, 関連する問題 [#2328](https://github.com/xuri/excelize/issues/2328)
* `SetCellFormula` 関数が通常の数式で上書きする際に共有数式メタデータをクリアしない問題を修正, 関連する問題 [#2337](https://github.com/xuri/excelize/issues/2337)

### パフォーマンスの最適化

* 暗号化されていないワークブックの読み取り時のメモリ使用量を削減し、行イテレーターのメモリ使用量を最大 85% 削減
* 座標変換および一部の内部関数のパフォーマンスを最適化
  * `ColumnNumberToName` 関数のメモリ割り当てを約 90% 削減
  * `CoordinatesToCellName` 関数のメモリ割り当てを約 13% 削減
  * 内部関数 `isNumeric` のメモリ割り当てを約 68% 削減
