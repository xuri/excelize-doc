# Excelize v2.11.0

{{ book.info }}

* 即将发布：2026年7月7日

## 版本说明

此版本中最显著的变化包括：

### 兼容性提示

* 升级 Go 语言版本要求至 1.25.0 或更高版本，以升级依赖包 `golang.org/x/net`
* 关于 `AddChart`、`AddChartSheet` 和 `AddShape` 函数的不兼容变更
  * `Chart` 数据类型中 `Title` 字段的数据类型由 `[]RichTextRun` 更改为 `ChartTitle`
  * `Shape` 数据类型中 `Line` 字段的数据类型由 `ShapeLine` 更改为 `LineOptions`
  * 数据类型 `ChartDashType` 重命名为 `LineDashType`
  * 数据类型 `ChartLineType` 重命名为 `LineType`
  * 数据类型 `ChartLine` 重命名为 `LineOptions`

### 安全更新

* 读取工作表时检查行号以防止无限制内存分配 (CVE-2026-54063)，相关安全公告 [#GHSA-h69g-9hx6-f3v4](https://github.com/xuri/excelize/security/advisories/GHSA-h69g-9hx6-f3v4)
* 通过验证工作表行号是否超出最大限制来防止攻击者控制流式读取内存分配 (CWE-400 和 CWE-770)，相关安全公告 [#GHSA-q5j5-6p94-4gwc](https://github.com/xuri/excelize/security/advisories/GHSA-q5j5-6p94-4gwc)
* 防止在共享字符串表索引无效时获取单元格值引发程序 panic (CWE-248 和 CWE-755)，相关安全公告 [#GHSA-fx5j-qcqg-grpf](https://github.com/xuri/excelize/security/advisories/GHSA-fx5j-qcqg-grpf)

### 新增功能

* 新增 `ChartTitle` 数据类型
* 新增 `PivotTableShowValuesAs` 数据类型
* 新增 `PivotTableShowValuesAsType` 枚举类型
* 新增 3 个导出错误变量：`ErrPivotTableShowValuesAsBaseField`、`ErrPivotTableShowValuesAsBaseItem` 和 `ErrUnsupportedPivotTableShowValuesAsType`
* `PivotTableField` 数据类型中新增 `ShowValuesAs` 字段
* `PivotTableOptions` 和 `SlicerOptions` 数据类型中新增 `SelectedItems` 字段
* 新增 `AutoFitColWidth` 函数，支持自动调整列宽，相关 issue [#92](https://github.com/xuri/excelize/issues/92)
* `SetSheetName` 函数支持对原本无效的名称工作表进行重命名，相关 issue [#548](https://github.com/xuri/excelize/issues/548)
* `CalcCellValue` 函数支持计算带有波浪号通配符条件的单元格公式
* `CalcCellValue` 函数支持计算带有隐式交集的单元格公式，并修复正则表达式条件计算有误的问题
* `CalcCellValue` 函数支持计算带有跨工作表的三维引用单元格公式，相关 issue [#2303](https://github.com/xuri/excelize/issues/2303)
* `CalcCellValue` 函数支持返回不应用数字格式的原始单元格公式计算结果，相关 issue [#2316](https://github.com/xuri/excelize/issues/2316)
* `AddPivotTable` 函数在创建数据透视表时，支持检查相同数据字段是否同时出现在数据透视表的列、行和筛选字段中，以防止生成损坏的工作簿
* `AddPivotTable` 和 `GetPivotTables` 函数支持设置和获取数据透视表数据字段的"值显示方式"，相关 issue [#2340](https://github.com/xuri/excelize/issues/2340)
* `AddPivotTable`、`AddSlicer`、`GetPivotTables` 和 `GetSlicers` 函数支持设置和获取数据透视表及数据透视表切片器的筛选选定项，相关 issue [#2154](https://github.com/xuri/excelize/issues/2154)
* `AddChart` 和 `AddChartSheet` 函数在添加图表时，支持使用公式设置图表标题
* `AddChart` 和 `AddChartSheet` 函数在添加图表时，支持设置图表标题和图表坐标轴标题的布局
* `AddChart` 和 `AddChartSheet` 函数在添加图表时，支持设置图表标题和图表坐标轴标题的线条格式
* `AddShape` 函数在添加形状时，支持设置形状的线条格式
* `NewStyle` 函数在创建纯色填充样式时，若未指定自定义颜色值，则支持自动设置前景色和背景色

### 兼容性提升

* 修复图表工作表在 Excel Online 和 KingSoft WPS&trade; Office 中不显示的问题，相关 issue [#2273](https://github.com/xuri/excelize/issues/2273)

### 问题修复

* 修复 v2.8.1 中引入的问题，解决 issue [#1383](https://github.com/xuri/excelize/issues/1383)
* 修复 `AddChart` 函数在添加图表时，若未指定系列填充颜色将引发程序 panic 的问题
* 修复 `CalcCellValue` 函数在某些情况下，计算带有无效行引用的单元公式时将引发程序 panic 的问题，相关 issue [#2266](https://github.com/xuri/excelize/issues/2266)
* 修复使用 `CalcCellValue` 函数时公式精确文本匹配无法匹配子字符串的问题
* 修复 `CalcCellValue` 函数计算 BINOM.INV、CHIINV、CHISQ.DIST、IMSECH、IMSQRT 和 NEGBINOM.DIST 公式函数时的计算精度问题
* 修复 `CalcCellValue` 函数在某些情况下对包含 FREQUENCY 公式函数计算结果有误的问题
* 修复 `CalcCellValue` 函数计算括号或运算符内包含错误值的公式时，返回错误计算结果的问题，解决 issue [#2344](https://github.com/xuri/excelize/issues/2344)
* 修复 `AddComment` 函数添加批注时，若作者已存在于作者列表时，为所有批注分配错误作者 ID 的问题，相关 issue [#2289](https://github.com/xuri/excelize/issues/2289)
* 修复因返回错误前未正确释放文件锁而导致潜在死锁的问题
* 修复部分情况下自定义数字格式中逗号未被正确格式化的问题，解决 issue [#2297](https://github.com/xuri/excelize/issues/2297)
* 修复列迭代器读取包含无效工作表 XML 的损坏工作簿时引发程序 panic 的问题，解决 issue [#2299](https://github.com/xuri/excelize/issues/2299)
* 修复调整包含外部引用的公式后，公式中的单引号丢失问题
* 修复部分情况下无法解密使用 ECMA-376 Agile 加密和 SHA-512 哈希算法保护的工作簿的问题，解决 issue [#2328](https://github.com/xuri/excelize/issues/2328)
* 修复 `SetCellFormula` 函数在使用普通公式覆盖共享单元格公式时，未清除共享公式元数据的问题，解决 issue [#2337](https://github.com/xuri/excelize/issues/2337)

### 性能优化

* 降低读取未加密工作簿时的内存占用，行迭代器内存开销相较于上一版本降低最高约 85%
* 优化坐标转换函数及部分内部函数的性能
  * `ColumnNumberToName` 函数的内存开销相较于上一版本降低最高约 90%
  * `CoordinatesToCellName` 函数的内存开销相较于上一版本降低最高约 13%
  * 内部函数 `isNumeric` 的内存开销相较于上一版本降低最高约 68%
